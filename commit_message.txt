🛡️ Sentinel: [HIGH] Fix SSRF and Prototype Pollution in MCP proxy endpoint

🚨 Severity: HIGH
💡 Vulnerability: The `/api/mcp` endpoint in `gui/backend/server.js` used unvalidated property access (`mcpUrls[mcp]`) and only checked for a falsy value (`!mcpUrl`). By passing a built-in object property like `__proto__` or `toString`, an attacker could bypass the check and cause Server-Side Request Forgery or Application crash.
🎯 Impact: An attacker could bypass the `Unknown MCP endpoint` validation, causing unhandled exceptions when passing built-in functions to Axios, or potentially discovering and exploiting internal network structure.
🔧 Fix: Used `typeof mcp === 'string'` and `Object.prototype.hasOwnProperty.call()` to safely validate that the requested key actually exists on the `mcpUrls` object itself, and verified the fetched URL is a string.
✅ Verification: Ran a custom fetch test to submit `mcp: "__proto__"` and verified it now securely returns `400 Bad Request` with `Unknown MCP endpoint`.
